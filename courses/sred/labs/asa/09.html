A09 - High Availability

Topology

Tasks

1. Primary ASA Setup

In this task you will configure the primary ASA of a high-availability setup.

A. Hostname

Configure the hostname of the first ASA (note the title of the console window) to ASA1. ASA1 will play the role of the primary active device.

B. Failover interface

Configure interface Gi2 as a failover interface. Use Control for the interface name.

Show solution
Hide solution
Show solution
ciscoasa(config)# hostname ASA1
ASA1(config)# failover lan interface Control gigabit 2
INFO: Non-failover interface config is cleared on Ethernet0/3 and its sub-interfaces

Use the show running interface Gi2 command to inspect the automatically generated commands for this interface. Bring Gi2 up.

Show solution
Hide solution
Show solution
ASA1# show running-config interface gigabit 2
!
interface Gigabit2
 description LAN Failover Interface
 shutdown
ASA1(config)# interface gigabit 2
ASA1(config-if)# no shutdown

C. Inside interface

Configure Gi0 using the following settings:

  • name: inside
  • security level: 100
  • IP address: 192.168.1.1/24
  • Standby address: 192.168.1.2
Show solution
Hide solution
Show solution
ASA1(config)# interface gigabit 0
ASA1(config-if)# nameif inside
INFO: Security level for "inside" set to 100 by default.
ASA1(config-if)# ip address 192.168.1.1 255.255.255.0 standby 192.168.1.2
ASA1(config-if)# no shutdown

D. Outside interface

Configure Gi1 using the following settings:

  • name: outside
  • security level: 0
  • IP address: 141.85.99.1/24
  • Standby address: 141.85.99.2
Show solution
Hide solution
Show solution
ASA1(config)# interface gigabit 1
ASA1(config-if)# nameif outside
INFO: Security level for "outside" set to 0 by default.
ASA1(config-if)# ip address 141.85.99.1 255.255.255.0 standby 141.85.99.2
ASA1(config-if)# no shutdown

E. Failover interface addressing

Configure the 11.11.11.1/30 failover IP address for the active unit and the 11.11.11.2 IP address for the standby unit.

Show solution
Hide solution
Show solution
ASA1(config)# failover interface ip Control 11.11.11.1 255.255.255.252 standby 11.11.11.2

F. Primary failover unit

Configure ASA1 as the primary failover unit.

Show solution
Hide solution
Show solution
ASA1(config)# failover lan unit primary

G. Activate failover

Activate the failover process using the failover command.

Show solution
Hide solution
Show solution
ASA1(config)# failover

3. HA Monitoring

In this task you will learn how to monitor the active ASA in a high-availability setup.

A. ICMP inspection

Configure ICMP inspection on the active ASA. Check if the configuration has been replicated to the standby ASA.

Show solution
Hide solution
Show solution
ASA1(config)# access-list SELECT_ICMP extended permit icmp any any
ASA1(config)# class-map CLASS_ICMP
ASA1(config-cmap)# match access-list SELECT_ICMP
ASA1(config)# policy-map POLICY_ICMP
ASA1(config-pmap)# class CLASS_ICMP
ASA1(config-pmap-c)# inspect icmp
ASA1(config)# service-policy POLICY_ICMP interface inside
! On "ASA2":
ASA1# show running-config class-map
!
class-map CLASS_ICMP
 match access-list SELECT_ICMP
!

B. Setup remote access

Configure the telnet and enable passwords on each router (not the ASA).

Show solution
Hide solution
Show solution
R1(config)#enable secret cisco
R1(config)#line vty 0 4
R1(config-line)#password cisco
R1(config-line)#login
R2(config)#enable secret cisco
R2(config)#line vty 0 4
R2(config-line)#password cisco
R2(config-line)#login

C. Ping!

Start 1000-packet ping from R1 to R2.

Show solution
Hide solution
Show solution
