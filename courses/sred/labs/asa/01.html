2. Basic IP Addressing

In this task you will configure IP addresses on ASA firewall interfaces.

A. IP Addressing

Configure the following IP addresses on the specified interfaces.

ASA interfaces are shutdown by default. Don't forget to issue the no shutdown command to bring them up.

  • Gi0: 141.85.99.1/24
    Rezolvare
    Atlas(config)# interface gigabit 0
Atlas(config-if)# no shutdown
Atlas(config-if)# ip address 141.85.99.1 255.255.255.0
  • Gi1: 10.10.10.1/24
    Rezolvare
    Atlas(config-if)# interface gigabit 1
Atlas(config-if)# no shutdown
Atlas(config-if)# ip address 10.10.10.1 255.255.255.0
  • Gi2: 192.168.1.1/24
    Rezolvare
    Atlas(config-if)# interface gigabit 2
Atlas(config-if)# no shutdown
Atlas(config-if)# ip address 192.168.1.1 255.255.255.0

Print a summary of current interface status. Why are the configured IP addresses missing? Do the issued commands appear in running-config?

Rezolvare
Atlas# show interface ip brief
Interface                  IP-Address      OK? Method Status                Protocol
Gigabit0                   unassigned      YES manual up                    up
Gigabit1                   unassigned      YES manual up                    up

B. Interface names

Configure a name for each interface. Use inside, dmz and outside for names.

Rezolvare
Atlas(config)# interface gigabit 0
Atlas(config-if)# nameif outside
INFO: Security level for "outside" set to 0 by default.
Atlas(config-if)#
Atlas(config-if)# interface gigabit 1
Atlas(config-if)# nameif dmz
INFO: Security level for "dmz" set to 0 by default.
Atlas(config-if)#
Atlas(config-if)#
Atlas(config-if)# interface gigabit 2
Atlas(config-if)# nameif inside
INFO: Security level for "inside" set to 100 by default.

Print the interface names and security levels without using the show running-config command.

Rezolvare
Atlas# show nameif
Interface                Name                     Security
Gigabit0                 outside                    0
Gigabit1                 dmz                        0
Gigabit2                 inside                   100

Configure a security level of 50 on the DMZ interface.

Rezolvare
Atlas(config)# interface gigabit 1
Atlas(config-if)# security-level 50

Print all the security levels using the show running-config command. Use an include filter to only display the relevant lines.

Rezolvare
Atlas# show running-config | include security-level
 security-level 0
 security-level 50
 security-level 100
 no security-level
 no security-level
 no security-level

Display all the commands configured on interface Gi2 without using filters.

C. Outside telnet access

Permit telnet access on the outside interface only from the 141.85.99.0/24 network. Does it work? Why not?

Rezolvare
Atlas(config)# telnet 141.85.99.0 255.255.255.0 outside
 
R1#telnet 141.85.99.1
Trying 141.85.99.1 ...
% Connection timed out; remote host not responding
Snippet from Cisco documentation:
Note: You can enable Telnet to the security appliance on all interfaces. However, the security appliance enforces that all Telnet traffic to the outside interface be protected by IPsec. In order to enable a Telnet session to the outside interface, configure IPsec on the outside interface to include IP traffic that is generated by the security appliance and enable Telnet on the outside interface.

Note: In general, if any interface that has a security level of 0 or lower than any other interface, then PIX/ASA does not allow Telnet to that interface.

Note: It is not recommended to access the security appliance through a Telnet session. The authentication credential information, such as password, is sent as clear text. The Telnet server and client communication happens only with the clear text. Cisco recommends to use SSH for a more secured data communication.

D. SSH access

Configure SSH version 2 access. Only allow hosts from the 141.85.99.0/24 network. Does it work? Why?

With the release of ASA 8.4, it is no longer possible to login using the default user pix. You will need to configure local AAA authentication and a new user. See the Release Notes for more information. 

Atlas(config)# aaa authentication ssh console LOCAL
Atlas(config)# username student password student

Rezolvare
Atlas(config)# crypto key generate rsa modulus 1024
INFO: The name for the keys will be: <Default-RSA-Key>
Keypair generation process begin. Please wait...
Atlas(config)# ssh 141.85.99.0 255.255.255.0 outside
Atlas(config)# ssh version 2

R1#ssh -l pix 141.85.99.1

Password: class

Atlas>
