F10 - Practice Exam

Sample exam from 2013

Topology

Tasks

1. Basic IP Addressing (25p)

a. (5p) On FG1, configure the 200.0.0.10/24 IP address on the port2 interface.

b. (5p) Configure the following loopback interfaces:

  • FG1: Loopback 0
    • IP Address: 11.11.11.11/32
  • FG2: Loopback 0
    • IP Address: 22.22.22.22/32

c. (10p) On FG1, configure a new DHCP server on port2 for the 200.0.0.1 - 200.0.0.60 addresses. Exclude the 200.0.0.1 and 200.0.0.20 addresses from the pool. The pool should also supply the 200.0.0.1 default gateway.

d. (5p) On FG2, configure the port2 interface to take its IP address via DHCP. FG2 should also take its default route via DHCP.

2. Users and policies (20p)

a. (5p) On FG1, create a new super_admin type administrator called SRED_ADMIN with password sred. Login using the new account.

b. (15p) On FG1, create a new firewall policy with authentication that allows only ICMP and HTTP traffic. The policy must perform NAT. Use the following settings (create the appropriate objects when required):

Internal network name INTERNAL
Service group name WB_ICMP
Timetable name SRED_EXAM
User/password SRED/SRED
User group SRED_GROUP

c. (5p) On FG1 and FG2, create two new firewall policies that permit ICMP traffic both ways between the port2 to the Loopback0 interfaces.

3. Routing (15p)

a. (10p) Establish connectivity between the two loopback interfaces using OSPF. Use the loopback IP addresses as router ids. The specific 11.11.11.11/32 and 22.22.22.22/32 must be listed in the routing table.

b. (5p) Using extended ping, test that FG1:Loopback0 can ping FG2:Loopback0.

4. Site to Site VPN (30p)

a. (20p) Establish a route-based site to site VPN between FG1 and FG2. Remember that FG2:port2 took its address via DHCP. Use the following settings:

  • Inside (private) network: Loopback0
  • Outside (public) network: port2
  • Phase 1 tunnel name: tun1
  • Phase 2 tunnel name: tun2
  • Password: sred2013
  • Enable IPSec interface mode

b. (5p) Create the following static routes:

  • On FG1: 22.22.22.22/32 should be reached via the VPN tunnel
  • On FG2: 11.11.11.11/32 should be reached via the VPN tunnel

c. (5p) Using extended ping, test that FG1:Loopback0 can ping FG2:Loopback0.

5. Web filtering (20p)

A web server has been deployed with the 200.0.0.1 IP address. You can send requests to this server to test your solutions.

a. (10p) Block all HTTP GET requests for URLs containing the sred_exam_solution string.
