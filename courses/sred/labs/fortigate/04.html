F04 - Advanced Policies

Topology

Tasks

1. Basic Setup

In this task you will configure connectivity between your host the Internet Server.

The Internet Server is simulated via an Ubuntu Server Virtual Machine

Connect to the FortiGate using a web browser of your choice. The user is admin and no password is configured.

The IP address of the FortiGate is 10.99.99.10

Configure the 200.0.0.10/24 IP address on the port2 FortiGate interface. Test that the FortiGate can ping the Internet Server (200.0.0.1/24).

Add a default route on the FortiGate. Use the Ubuntu Server as gateway.

Configure a NAT policy on the FortiGate s.t. the host can reach the Internet server using any protocol. Test using SSH, HTTP and ICMP.

2. Application filtering

In this exercise you will block outbound FTP traffic.

Add a new policy rule that only blocks FTP traffic. Verify by running the ftp 200.0.0.1 command. HTTP access to the Ubuntu Server must still be allowed.

If you receive the following prompt, it means that ftp traffic is not blocked by the firewall: 

ftp: connect: Connection refused
ftp>

3. URL filtering

In this exercise, you will configure URL filtering based on regular expressions.

A. Create the URL filter

Create a new URL filter named MyURLFilter from the UTM Security Profiles / Web Filter / URL Filter tab.

Create a rule that blocks all URLs containing the .exe extension

B. Configure the web filter

Apply the newly create URL filter to the default web filter. Use the UTM Security Profiles / Web Filter / Profile tab.

C. Configure the policy

Enable web filtering on the current policy that permits HTTP traffic.

D. Test

Try to access the 200.0.0.1/myfile.exe page. If you get a 404 HTTP error, then the web filtering is not working properly.

4. Traffic Shaping

A. Upload a file

First, you need to generate a large file. You will do this using the dd command on the host computer: 

linux# dd if=/dev/urandom of=mylargefile.dat bs=1M count=64

Copy the large file to the Ubuntu Server using scp. Monitor the upload speed. 

linux# scp mylargefile.dat student@200.0.0.1:

B. Traffic shapers

Configure a Per-IP traffic shaper named MyTrafficShaper from the Firewall objects/Traffic Shaper/Per-IP tab.

Limit the bandwidth to 4096Kbit/second.

C. Configure the policy

Apply the traffic shaper to the policy that permits SSH traffic.

D. Test

Try to upload the large file again. Monitor the upload speed.

Bonus: Data Leak Prevention

Configure data leak prevention for HTTP messages containing the SRED text string. The required action is to drop offending packets.

For testing, you can create and access the following web page: 

<form name="input" action="test.html" method="post">
Username: <input type="text" name="user">
<input type="submit" value="Submit">
</form>
