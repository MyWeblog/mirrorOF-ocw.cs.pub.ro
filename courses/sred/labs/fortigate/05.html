F05 - Routing

Topology

Tasks

1. Connectivity Test

In this task you will test for IP connectivity between the virtual devices.

Use the topology above to discover the required IP addresses

We recommend you write down a diagram of the network containing IP addresses and interface names. It will be of great help to you throughout the lab.

Check that:

  • You can ping the outside (port2) interface of FortiGate2 from the outside (port2) interface of FortiGate1
  • You can ping the inside (port1) interface of FortiGate2 from the inside (port1) interface of FortiGate1
  • You can log in successfully on FortiGate1 and FortiGate2 from the host computer
  • You can ping the Ubuntu Server from FortiGate1 and FortiGate2

Traffic from the host computer to the Ubuntu Server is routed through FortiGate1.

2. RIP

In this task you will configure RIP on the two FortiGate firewalls.

A. Loopback interfaces

Configure the following loopback interfaces on the FortiGate devices:

  • On FortiGate1: 1.1.1.1/24
  • On FortiGate2: 2.2.2.2/24

Use the System/Network/Interface tab.

B. RIP Configuration

Configure RIP on both firewalls. Include the network between the Server and firewalls and the loopback networks. Do NOT include the 2.2.2.0/24 network on FortiGate1 and the 1.1.1.0/24 network on FortiGate2 (why?).

Use the Router/Dynamic/RIP tab.

C. Debugging

Check that the 1.1.1.0/24 and 2.2.2.0/24 routes are learned via RIP.

Use the Router/Monitor/Routing monitor tab.

Test that you can ping 1.1.1.1 from FortiGate2 and 2.2.2.2 from FortiGate1.

You will need to adjust the configured policies.

3. OSPF

In this task you will configure OSPF on the two FortiGate firewalls.

A. Loopback interfaces

Configure the following loopback interfaces on the FortiGate devices:

  • On FortiGate1: 11.11.11.11/24
  • On FortiGate2: 22.22.22.22/24

B. OSPF Configuration

Configure OSPF on both firewalls.

Configure the following router IDs:

  • On FortiGate1: 11.11.11.11
  • On FortiGate2: 22.22.22.22

Create area 0.0.0.0 on both firewalls.

Include the network between the Server and firewalls and the newly added loopback networks in area 0.0.0.0.

C. Debugging

Check that the 11.11.11.0/24 and 22.22.22.0/24 are learned via OSPF.

Test that you can ping 11.11.11.11 from FortiGate2 and 22.22.22.22 from FortiGate1.

4. Policy Routes

Create a policy route on FortiGate1 that forwards any UDP traffic coming from 10.99.99.1 to FortiGate2.

Traceroute uses UDP on Linux.

Test using traceroute.
