Upstream Challenge

Ce este Upstream Challenge?

  • pstream Challenge** este o competiție dedicată studenților (în special studenţilor cursului de SO) și constă în contribuții aduse proiectelor open source. Acest concurs se desfășoară în perioada [Februarie - Aprilie] și nu aduce puncte suplimentare în cadrul materiei de SO. Concursul este sponsorizat de Intel Romania.

Desfășurare

  • Start Date: 14 Februarie 2012
  • End Date: 13 Mai 2012
  • Premiere: 16 Mai 2012, la ora 19:00, în sala EC105, după cursul 13 de SO de la seria CC

Reguli

  • tot ce trebuie să faceți este să contribuiți la proiecte open source, urmăriți mai jos definiția pentru contribuție;
  • angajatii si internii Intel pot participa la concurs, insa nu sunt eligibili pentru a câștiga premiile și vor aparea intr-un clasament separat;
  • dupa ce o contributie a fost acceptata aceasta va fi anuntata pe lista de discutii intr-un mail cu subiectul [Upstream Challenge] patch title;
  • interactiunea cu studentii se va face pe lista normala de discutii;
  • evaluarea si premierea vor fi facute de echipa de SO pe baza contributiilor voastre;
  • premiul cel mare constă într-un Ultrabook Acer S3-951-2464G34iss oferit de Intel România în valoare de 3300 RON - disclaimer.

Ce înseamnă contribuție?

Contribuție este considerat orice patch acceptat într-un proiect open source, indiferent de gradul de dificultate (corecție de ortografie, actualizare de comentariu, rezolvarea unui bug, implementarea unei noi funcționalități). Pentru a vă face o imagine cât mai clară despre ce inseamnă contribuție, urmăriți link-urile de mai jos:

  • corecție ortografie - fs/eventpoll.c: fix spelling
  • actualizare comentariu necorespunzător - af_unix: update locking comment
  • rezolvarea unui bug - jit: Fix bug in insert_list
  • adăugare de noi funcționalități: move splitter patch for Kate

Quick start

  • alege un proiect open source
  • clonează sursele, compilează și ruleaza codul
  • abonează-te la lista de discuții
  • informează-te despre modelul de dezvoltare al proiectului
    • urmărește patch-urile trimise pe listă
    • urmărește bug-urile raportate și incearcă să le rezolvi
    • caută o listă de TODO-uri pentru proiectul respectiv
  • scrie cod
