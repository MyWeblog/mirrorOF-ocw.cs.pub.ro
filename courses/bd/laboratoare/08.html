Laboratorul 08.

Formatarea datelor de ieșire în SQL Plus

Comenzi SQL Plus

  • @ - execută un fișier de comenzi indirecte
  • / - execută comanda curentă din buffer
  • ACC[EPT] var [NUM[BER]/CHAR] [PROMPT text / NOPR[OMPT]][HIDE] - citeste o data de intrare si o stocheaza intr-o variabila
  • A[PPEND] text - adauga un text la sfarsitul liniei curente din buffer
  • BRE[AK] [ON report_element [action[action]]].. - specifica modul de formatare intr-un raport prin suprimarea afisarii inregistrarilor duplicate pe coloane, salt de linie, efectuarea de calcule cu comanda COMPUTE
  • BTI[TLE] [printspec [ text / variable] … ] | OFF / ON - plaseaza un text la sfarsitul raportului
  • C[HANGE] sepchar old [sepchar [new]] - schimba un text in linia de comanda
  • CL[EAR] option - reseteaza setarile facute cu alte comenzi, cum ar fi BREAKS, BUFFER, COLUMNS, COMPUTERS, SCREEN
  • COL[UMN] [(column / expr) [option …]] - specifica atribute de afisare pentru o coloana, cum ar fi specificarea antetului si formatul de afisare
  • COMP[UTE] [function OF (expr/column/alias) … ON (expr/column/alias/REPORT/ROW) - face anumite calcule pe coloane aplicand function
  • COPY FROM username [/password]@database_specification | TO username[/password]@database_specification (APPEND/CREATE/INSERT/REPLACE) destination_table [(col1,col2, …)] USING query - copiaza o tabela dintr-un user in alt user conform unui query specificat
  • DEF[INE] [variable]|[variable=text] - defineste o variabila a userului
  • DEL - sterge linia curenta din buffer
  • DESC[RIBE] [user.]object[@database_link_name] - afiseaza structura unei tabele, view sau synonym
  • DISCONNECT - deconecteaza userul curent dar nu iese din SQL Plus
  • EDIT [file_name[.ext]] - invoca un editor pentru comanda curenta din buffer
  • EXIT|QUIT - executa commit la ultimele tranzactii si iese din SQL Plus
  • GET file_name[.ext][LIST|NOTLIST] - incarca un fisier de comenzi indirecte in buffer
  • HELP [topic] - acceseaza documentatia din SQL Plus
  • HO[ST] [command] - revenire partiala in sistemul de operare fara a parasi SQL Plus
  • I[NPUT] [text] - adauga noi linii la comanda curenta din buffer
  • L[IST] [n m] [n LAST] [* LAST][LAST] - listeaza liniile comenzii din buffer incepand cu linia curenta (*) sau de la linia n la m (fara parametri listeaza ultima comanda din buffer) *PAU[SE] [text] - opreste afisarea pe ecran, afiseaza text si asteapta RETURN
  • PROMPT [text] - afiseaza un mesaj pe ecran
  • R[UN] - afiseaza si executa ultima comanda din buffer
  • SAV[E] file_name[.ext] [CREATE|REPLACE|APPEND] - salveaza comanda din buffer intr-un fisier
  • SET system_variable value seteaza parametri specifici sesiunii curente
  • SHO[W] option - afiseaza valorile variabilelor de sistem, unde option poate fi: ALL, BTITLE, NO, PNO, RELEASE, SPOOL, SQLCODE, TTITLE, USER
  • SPO[OL] file_name[.ext]|OFF|ON - rediecteaza rezultatele unei interogari spre un fisier
  • STA[RT] file_name[.ext][arg1,arg2,…] - executa un fisier de comenzi indirecte
  • TTI[TLE] [printspec [text|variable] …]|[OFF|ON] - plaseaza un text la inceputul raportului
  • UNDEF[INE] variable - reseteaza variabile definite in sesiunea curenta
Comanda SET primeste urmatorii parametri:
  • ARRAY[SIZE] (20/n) - specifica numarul maxim de linii din buffer rezultate in urma unei interogari, iar n poate fi maxim 5000
  • DEF[INE] ('&'|c|OFF|ON) - seteaza promptul pentru variabile substituite
  • ECHO[OFF|ON]) - controleaza cum sunt afisate comenzile executate dintr-un fisier de comenzi (OFF suprima acest lucru)
  • FEED[BACK] (6|n|OFF|ON) - afiseaza numarul de inregistrari returnate de un query cand sunt returnate cel putin n inregistrari
  • HEA[DING] (OFF|ON) - controleaza daca sunt afisate aliasurile de coloane
  • LIN[ESIZE](80|n) - seteaza numarul de caractere care vor fi afisate pe o linie si de asemenea controleaza centrarea si aliniamentul din dreapta a textului. Valoarea maxima pentru n este 500
  • NEWP[AGE](1|n) - seteaza numarul liniilor goale ce vor fi tiparite intre pagini (titlul de jos de la fiecare pagina si titlul de sus al paginii urmatoare)
  • NUMF[ORMAT] format - seteaza formatul implicit pentru afisarea valorilor numerice
  • NUM[WIDTH](10|n) - seteaza lungimea implicita pentru afisarea valorilor numerice
  • PAGES[IZE] (24|n) - seteaza numarul liniilor pe o pagina
  • PAU[SE] (OFF|ON|text) - permite oprirea afisarii pe ecran in timpul unei interogari
  • SHOW[MODE] (OFF|ON) - permite afisarea variabilelor de sistem cand sunt schimbate cu comanda SET
  • SPA[CE] (1|n) - seteaza numarul de spatii dintre coloane in timpul afisarii (valoarea maxima pentru n este 10)
  • SQLCASE (MIXED|LOWER|UPPER) - permite convertirea comenzilor SQL inainte de executie. LOWER/UPPER convertesc comenzile iar MIXED va lasa textul nemodificat
  • VER[IFY] (OFF|ON) - pentru ON afiseaza textul unei comenzi inainte si dupa substituirea variabilei cu valoarea sa
  • TIM[ING] (OFF|ON) - pentru ON, SQL Plus va da statistici de timp la fiecare comanda executata
  • TERM[OUT] (OFF|ON) - controleaza afisarea iesirii generate de comenzile executate dintr-un fisier de comenzi. Pentru OFF, iesirea poate fi prelucrata fara sa apara pe ecran.
Exemplu
SET LINES 120
SET PAGESIZE 24
SET SPACE 2
SET NEWPAGE 5
SET HEADING OFF
SET FEEDBACK OFF
Comanda COLUMN poate primi urmatoarele optiuni:
  • ALI[AS] alias - insereaza un alias
  • CLE[AR] - reseteaza optiunile anterioare
  • FOLD_A[FTER] - insereaza CR dupa antet si dupa fiecare rand
  • FOLD_B[EFORE] - insereaza CR dupa antet si inainte fiecarui rand
  • FOR[MAT] format - specifica formatul de afisare An pentru coloane alfanumerice sau unul din formatele numerice
  • HEA[DING] text - defineste antetul coloanei
  • JUS[TIFY] L[EFT]|C[ENTER]|R[IGHT] - specifica aliierea antetului (implicit dreapta pentru coloane numerice si stanga pentru celelalte tipuri)
  • LIKE (expr/alias) - copiaza setarile altei coloane definite anterior
  • NEWL[INE] - salt la linie noua
  • NEW_V[ALUE] variable - specifica o variabila care va stoca valoarea coloanei
  • NOPRI[NT] |PRI[NT] - specifica daca coloana va fi afisata
  • NULL - specifica ce text va fi afisat pentru o valoare nula
  • ON|OFF - controleaza starea setarilor pentru o coloana
  • WRA[PPED]|[WOR[D_WRAPPED]|TRU[NCATED] - specifica cum va fi afisat un sir prea lung pentru o coloana
Exemplu
COL id_dep FOR 099 HEA 'DEPARTAMENT' JUSTIFY CENTER
COL functie FOR A10 HEA 'JOB' JUSTIFY LEFT
COL id_ang FOR 9999 HEA 'ECUSON' JUSTIFY CENTER
COL salariu FOR 99,999
COL comision FOR 99,999.99 NULL '0'
COL venit FOR 99,999.99 HEA 'VENIT LUNAR'
Comenzile BTITLE si TTITLE primesc urmatoarele optiuni:
  • COL n - se pozitioneaza pe a n-a linie (daca pozitia a fost trecuta, se intoarce pe pozitia specificata)
  • SKIP n - se executa salt peste n linii (daca n=0, se pozitioneaza la inceputul liniei curente)
  • TAB n - se executa salt peste n coloane (daca n este negativ, se executa salt inapoi)
  • LEFT - se pozitioneaza articolul care urmeaza acestei optiuni in partea stanga a liniei. Daca nu mai exista nicio optiune de aliniere in comanda, toate articolele vor fi aliniate in ordinea aparitiei in partea stanga, altfel se aliniaza numai cele care apar pana la urmatoarea optiune.
  • CENTER - in acest caz, pozitionarea se face central si se ia in calcul lungimea liniei setata cu LINESIZE
  • RIGHT - similar cu optiunea de mai sus, dar pozitionarea se face la dreapta
  • BOLD - se specifica ca afisarea se face cu caractere ingrosate
  • FORMAT char - se specifica formatul de afisare pentru caractere numerice sau alfanumerice
  • UNDERLINE - se specifica ca afisarea se face cu caractere subliniate
Comenzile BREAK si COMPUTE

Comanda BREAK este folosita pentru fragmentarea unui raport in mai multe segmente, iar comanda COMPUTE executa anumite calcule pe segmentele respective. Un raport poate fi fragmentat dupa:

  • Coloana
  • Expresie
  • Rand (ROW)
  • Raport (REPORT)

si se pot executa urmatoarele actiuni in urma fragmentarii

  • Salt de n randuri (SKIP n)
  • Salt de pagina (SKIP PAGE)
  • Duplicarea sau neduplicarea valorilor pe coloane (DUPLICATES | NODUPLICATES)

Calculele care se pot face cu comanda COMPUTE sunt:

  • AVG - calcul medie (pentru date de tip number)
  • COU[NT] - numara valorile nenule (pentru orice tip de date)
  • MAX[IMUM] - valoare maxima (pentru date de tip number si char)
  • MIN[IMUM] - valoare minima (pentru date de tip number si char)
  • NUM[BER] - numara randuri (pentru orice tip de data)
  • STD - calcul deviatie standard valori nenule (pentru date de tip number)
  • SUM - calcul suma (pentru date de tip number)
  • VAR[IANCE] - calcul variatie (pentru date de tip number)

Exemplu: 

SET PAGES 30
COL A HEA 'DEPARTAMENT'
COL B FORMAT A25 HEA 'NUME/PRENUME'
COL C FORMAT A15 HEA 'JOB' JUSTIFY LEFT
COL D FORMAT 99,999 HEA 'SALARIU'
 
TTITLE LEFT 'PAG:' SQL.PNO CENTER UNDERLINE 'LISTA SALARII'
BTITLE R 'DIRECTOR'
 
BREAK ON A NODUP ON REPORT
COMP SUM OF D ON A SKIP 1 REPORT
 
SELECT d.den_dep A, a.nume B, a.functie C, salariu D
FROM angajati a, departamente d
WHERE a,id_dep=d.id_dep
ORDER BY a.id_dep
/
TTITLE OFF
CLEAR COL

Exerciții

Exercițiile nu sunt identice între 2 ședințe de laborator. Diferențele sunt date de parametrii adiționali prezenți în fiecare dintre ele, ce vor fi descriși de laborant în cadrul fiecărui laborator.

1. Scrieti o cerere care afiseaza lista cu structura organizatorica a firmei, returnand campurile: departament, numele sefului din departamentul respectiv, numele subalternilor sefului si parametri aditionali. Titlul listei va fi 'STRUCTURA ORGANIZATORICA', se va face numerotarea paginii in partea stanga sus, data va fi data sistemului si se va pozitiona in partea dreapta sus a paginii, se va face total de salariu pe fiecare departament si un total general. Denumirile de departamente si numele sefilor nu vor fi duplicate, iar in partea de jos a paginii va aparea textul 'DIRECTOR RESURSE UMANE'. Se vor face setarile necesare si resetarile la sfarsit.

2. Scrieti o cerere care afiseaza o lista de premiere pentru toti sefii si pentru angajatii cu vechimea cea mai mare in fiecare departament. Prima se va calcula ca fiind numarul intreg de luni lucrate in firma + parametri aditionali. Cererea va returna: numele departamentului, numele angajatului, functia, data angajarii si prima. Titlul listei va fi 'LISTA DE PRIME', se va face numerotarea paginii in partea stanga sus, data va fi data sistemului si se va pozitiona in partea stanga jos a paginii, se vor face totaluri de salariu pe fiecare departament si un total general. Denumirile departamentelor nu vor fi duplicate, iar in partea de jos a paginii va aparea textul 'DIRECTOR ECONOMIC'. Se vor face setarile necesare si resetarile la sfarsit.

3. Scrieti o cerere care afiseaza fiecare angajat care are cel putin un subordonat (i.e. fiecare manager) impreuna cu subordonatii sai, salariile fiecaruia si parametri aditionali. Titlul listei va fi 'ECHIPE', se va face numerotarea paginii in partea stanga sus, data va fi data sistemului si se va pozitiona in partea stanga jos a paginii, se vor face totaluri de salariu pe fiecare echipa si un total general. Se va afisa de asemenea numarul persoanelor din fiecare echipa si totalul general. Denumirile sefilor nu vor fi duplicate, iar in partea de jos a paginii va aparea textul 'DIRECTOR PROIECTE'. Se vor face setarile necesare si resetarile la sfarsit.

4. Scrieti o cerere care afiseaza statistica functiilor din firma, returnand campurile: functie, numele persoanelor care ocupa functia respectiva, salariul lor si parametri aditionali. Titlul listei va fi 'STATISTICA FUNCTIILOR', se va face numerotarea paginii in partea stanga sus, data va fi data sistemului si se va pozitiona in partea dreapta sus a paginii, se va face media de salariu pe fiecare functie si se va afisa numarul persoanelor care ocupa functia respectiva. Denumirile functiilor nu vor fi duplicate, iar in partea de jos a paginii va aparea textul 'DIRECTOR RESURSE UMANE'. Se vor face setarile necesare si resetarile la sfarsit.
