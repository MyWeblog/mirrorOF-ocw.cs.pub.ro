2. Traffic policies

In this task you will configure specific traffic policies for each context.

A. London context

ICMP traffic from inside to outside should be permitted. You will need to apply the policy on the inside interface.

Show solution
Hide solution
Show solution
Atlas/London(config)# access-list SELECT-ICMP extended permit icmp any any
Atlas/London(config)# class-map CLASS-ICMP
Atlas/London(config-cmap)# match access-list SELECT-ICMP
Atlas/London(config)# policy-map INSPECT-ICMP
Atlas/London(config-pmap)# class CLASS-ICMP
Atlas/London(config)# service-policy INSPECT-ICMP interface inside
R1#ping 141.85.30.100
 
Type escape sequence to abort.
Sending 5, 100-byte ICMP Echos to 141.85.30.100, timeout is 2 seconds:
!!!!!
Success rate is 100 percent (5/5), round-trip min/avg/max = 16/36/84 ms

All source addresses in the London context should be translated using PAT. Use the IP address of the outside interface.

B. Paris context

Only HTTP traffic should be permitted to flow from inside to outside.

Show solution
Hide solution
Show solution
Atlas/Paris(config)# access-list ALLOW-ONLY-HTTP extended permit tcp any any eq 80
Atlas/Paris(config)# access-list ALLOW-ONLY-HTTP extended deny ip any any
Atlas/Paris(config)# access-group ALLOW-ONLY-HTTP out interface outside
R4(config)#ip http server
R2#telnet 141.85.40.100
Trying 141.85.40.100 ...
% Connection refused by remote host
 
R2#telnet 141.85.40.100 www
Trying 141.85.40.100, 80 ... Open

All inside addresses must be translated using PAT. Use the IP address of 141.85.40.101 as mapped address.
