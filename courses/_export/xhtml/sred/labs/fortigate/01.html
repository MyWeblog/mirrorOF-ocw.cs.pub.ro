Host Internet Connectivity

In this task you will learn how to configure Internet connectivity for hosts behind FortiGate firewalls.

On the FortiGate device, configure the 150.0.0.1/24 address on port2.

On the FortiGate device, configure a DHCP service on interface port2 using the 150.0.0.100/24 - 150.0.0.149/24 address space. The FortiGate must also provide a default route to its clients.

On Server, request an IP address from the FortiGate device using DHCP.

Display the leased addresses, both from the GUI and from the Console.

Rezolvare
Rezolvare

You can use the sudo dhclient eth0 command on the server to request an IP address via DHCP.

Test Internet connectivity again. Does it work? Why not?

Traffic will not pass through a FortiGate device without an explicit policy which allows it. You can view the implicit deny all policy under the Policy tab.

Create a firewall policy that permits any LAN device to access the Internet. The equipment will perform NAT.

Test Internet connectivity by pinging IP address 10.99.99.1.
