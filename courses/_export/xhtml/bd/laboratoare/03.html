Laboratorul 03

Metode de JOIN

Descriere generală

Pentru extragerea datelor din mai multe tabele din baza de date, comanda SELECT folosește una sau mai multe metode de JOIN. Sintaxa pentru un join simplu este următoarea: 

SELECT [DISTINCT,ALL] [table_alias].col expr_alias, ...
FROM [schema.table1] table1_alias,[schema.table2] table2_alias, ...
WHERE table1_alias.col=table2_alias.col
ORDER BY expression(POSITION)[ASC,DESC]

Parametrii comenzii au următoarea semnificație (cei din paranteze sunt opționali):

  • DISTINCT - returnează numai o înregistrarea în cazul în care comanda găsește rânduri duplicate
  • ALL - returnează toate înregistrările simple și duplicate (selectează de asemenea toate coloanele tabelelor din clauza FROM)
  • schema.table - reprezintă schema de identificare a tabelei (view-lui) specificată prin user.table_name
  • expr_alias - este un nume alocat unei tabele (view) care va fi folosit în cereri corelate
  • WHERE condition - reprezintă o clauză (înlănțuire de condiții) care trebuie să fie îndeplinită în criteriul e selecție a înregistrărilor
  • ORDER BY expression(position) - ordonează înregistrările selectate după coloanele din expresie sau în ordinea coloanelor specificate prin poziție

Tipuri de join

Equi Join
SELECT a.id_dep,b.den_dep departament, a.nume, a.functie
FROM angajati a, departamente b
WHERE a.id_dep=b.id_dep AND a.id_dep=10;
Non Equi Join
SELECT a.nume, a.salariu, b.grad
FROM angajati a, grila_salar b
WHERE a.salariu BETWEEN b.nivel_inf AND b.nivel_sup AND a.id_dep=20;
Self Join
SELECT a.nume nume_ang, a.functie functie_ang, b.nume nume_sef, b.functie functie_sef
FROM angajati a,angajati b
WHERE a.id_sef =b.id_ang AND a.id_dep=10;
Join Extern
SELECT a.id_dep,a.den_dep,b.nume,b.functie
FROM departamente a, angajati b
WHERE a.id_dep=b.id_dep(+);

Semnul (+) se pune în dreptul tabelei deficitare ca informații, pentru a afișa întotdeauna informațiile din cealaltă tabelă, chiar dacă nu au corespondent pentru join.

Join Vertical
SELECT id_dep,nume,functie,salariu
FROM anngajati
WHERE id_dep=10
UNION
SELECT id_dep,nume,functie,salariu
FROM angajati
WHERE id_dep=30;

Operatori pentru join vertical:

  • UNION pentru reuniune; UNION ALL pentru a păstra duplicatele;
  • INTERSECT pentru intersecție;
  • MINUS pentru diferență;

Exerciții

Exercițiile nu sunt identice între 2 ședințe de laborator. Diferențele sunt date de parametrii adiționali prezenți în fiecare dintre ele, ce vor fi descriși de laborant în cadrul fiecărui laborator.

1. Salarii și comisioane

Scrieți o cerere care selectează toți angajații (nume, funcție, salariu, comision, venit total, parametri adiționali) și informații despre departamentul în care lucrează (denumire departament, manager departament), unde fie numele angajatului, fie funcția lui, fie numele departamentului conțin un text dat de utilizator. Salvați cererea într-un fișier și apelați fișierul cu textul folosit în cerere ca argument.

2. Indemnizația de vechime

Modificați tabelul angajați să conțină o coloană denumită vechime, în ani (întregi) de la data angajării. Scrieți o cerere care selectează toți angajații (nume, funcție, salariu) împreună cu un câmp numit spor de vechime, calculat după următoarea formulă:

  • Întâi se calculează vechimea managerului de departament (vechimea departamentului Vd)
  • Apoi se calculează diferența între vechimea fiecărui angajat Va și vechimea departamentului D=Va-Vd
  • parametri adiționali
3. Pe cine dăm afară

Scrieți o cerere care selectează angajații, în ordinea descrescătoare a următoarelor criterii:

  • “ne costă prea mult”: salariu + comision
  • “deja se plictisesc”: vechime
  • “sunt din departamente condamnate, oricum”: vechimea managerului de departament
  • parametri adiționali

Evaluarea performanței cererilor

Putem evalua performanța cererilor noastre SQL cu ajutorul unei unelte prezente în majoritatea motoarelor de baze de date, anume EXPLAIN. Aceasta se folosește în modul următor: 

EXPLAIN PLAN SET statement_id='my_statement_id' FOR
Cererea_mea_SQL

Apoi, puteți afișa descrierea ultimului plan salvat cu următoarea cerere: 

SELECT plan_table_output FROM TABLE(dbms_xplan.display('plan_table',NULL,'serial'));

sau 

SELECT operation,"timestamp",search_columns,cost,cardinality,bytes FROM plan_table WHERE statement_id='my_statement_id';

În caz că tabelul PLAN_TABLE nu este deja creat, îl puteți crea cu următoarea cerere: 

CREATE TABLE PLAN_TABLE (
        STATEMENT_ID VARCHAR2(30),
"TIMESTAMP" DATE,
REMARKS VARCHAR2(80),
OPERATION VARCHAR2(30),
OPTIONS VARCHAR2(30),
OBJECT_NODE VARCHAR2(128),
OBJECT_OWNER VARCHAR2(30),
OBJECT_NAME VARCHAR2(30),
OBJECT_INSTANCE NUMBER,
OBJECT_TYPE VARCHAR2(30),
OPTIMIZER VARCHAR2(255),
SEARCH_COLUMNS NUMBER,
ID NUMBER,
PARENT_ID NUMBER,
"POSITION" NUMBER,
COST NUMBER,
CARDINALITY NUMBER,
BYTES NUMBER,
OTHER_TAG VARCHAR2(255),
OTHER LONG
);
