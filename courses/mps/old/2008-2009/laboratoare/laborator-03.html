Împărțirea sarcinilor (30 minute)

WBS (15 minute)

* Work Breakdown Structure http://en.wikipedia.org/wiki/Work_breakdown_structure * modul de construire * decomposition coding scheme (1.2.3, 1.3.1.2) * 100% rule * non-overlapping * utilitare http://dotproject.net/ http://www.criticaltools.com/wbsmain.htm (poate fi folosit cu Microsoft Project) === Gantt (10 minute) === * diagrame Gantt http://en.wikipedia.org/wiki/Gantt_chart exemplu: http://upload.wikimedia.org/wikipedia/en/0/0c/Ganttproject-house-building-sample.png * summary elements + terminal elements (asociere cu WBS) * dependinte, start-date, stop-date * utilitare Microsoft Project http://ganttproject.biz/ === Trac (10 minute) === * web-based software project management * http://trac.edgewall.org/ * wiki - http://trac.edgewall.org/wiki/TracWiki CamelCase timeline - http://trac.edgewall.org/wiki/TracTimeline * source browser - http://trac.edgewall.org/wiki/TracBrowser * tichete (bug reports) - http://trac.edgewall.org/wiki/TracTickets * roadmap - http://trac.edgewall.org/wiki/TracRoadmap * pluginuri - http://trac.edgewall.org/wiki/PluginList == Împărțire sarcini player audio-video (35 de minute) == * Împărțiți-vă în două echipe. (15 minute) Fiecare echipă va crea o structură de sarcini (de preferat WBS) pentru un player audio-video. În cadrul fiecărei sarcini va trebui să se precizeze * subsarcinile și suprasarcina asociate (se va observa, probabil, din figură) * dificultatea acesteia * dependințele față de alte sarcini * numărul de persoane alocate sarcinii (eventual oameni-oră) * durata sarcinii * inputul și outputul sarcinii * Completați în Trac-ul asociat proiectului vostru (pentru fiecare echipă) (10 minute) O pagină separată de wiki în format CamelCase de forma (Lab4Team1) Informații despre componența echipei: nume, prenume, grupă, adresă e-mail. Lista de sarcini ca mai sus. Creați 3-4 timeline-uri pentru câteva din sarcinile voastre în Trac. Trimiteți un bug report de test. * Discuție despre sarcinile fiecărei echipe (10 minute)
