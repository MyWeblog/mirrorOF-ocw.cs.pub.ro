  • Dispozitivele hardware sunt accesate prin intermediul unor fişiere din:
    1. /sys
    2. /proc
    3. /dev
    4. /media
    • [3]
    • (1)
  • Care din următoarele NU este un tip de fişier?
    1. bloc
    2. rețea
    3. pipe
    4. caracter
    • [2]
    • (1)
  • Care este caracteristica device-urilor caracter?
    1. volum mic de date
    2. viteză mare
    3. căutări frecvente
    4. organizare pe blocuri
    • [1]
    • (1)
  • Identificatorul major se referă la tipul dispozitivului. Identificatorul minor se referă la versiunea driver-ului.
    1. adevărat/adevărat
    2. adevărat/fals
    3. fals/adevărat
    4. fals/fals
    • [2]
    • (1)
  • Un dispozitiv de tip caracter este reprezentat de structura
    1. file
    2. inode
    3. cdev
    4. chrdev
    • [3]
    • (1)
  • Atunci când se accesează un pointer din user-space
    1. trebuie folosite numai funcţii speciale
    2. procesul NU se poate bloca
    3. se pot face doar scrieri
    4. este generat un oops; kernel-ul nu poate accesa user-space
    • [1]
    • (1)
  • Structura file
    1. este reprezentarea statică a unui inode
    2. conţine o referinţă la dispozitivul caracter
    3. este într-o relaţie 1:1 cu un fişier
    4. este folosită pentru a referi date specifice unui fişier deschis
    • [4]
    • (1)
  • La read şi write:
    1. sensul transferului este cel văzut din user-space
    2. NU se pot efectua transferuri parţiale
    3. poziţia în fişier este actualizată automat
    4. succesul este semnalat prin întoarcerea valorii 0
    • [1]
    • (1)
  • ioctl este un mecanism:
    1. de control
    2. de sincronizare
    3. accesibil numai din kernel
    4. care NU permite transferul de date
    • [1]
    • (1)
  • O coadă de aşteptare
    1. foloseşte busy-waiting
    2. NU permite folosirea unui timeout
    3. NU afectează rularea user-space
    4. permite blocarea proceselor în aşteptarea unei condiţii
    • [4]
    • (1)
